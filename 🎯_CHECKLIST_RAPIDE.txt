❌ SI VOUS RENCONTREZ DES PROBLÈMES:
════════════════════════════════════════════════════════════════════════════════

PROBLÈME: Port 3001 déjà utilisé
─────────────────────────────────
Solution 1: Trouver le process
  netstat -ano | findstr :3001

Solution 2: Tuer le process
  taskkill /PID <PID> /F
  (remplacez <PID> par le numéro trouvé)

Solution 3: Relancer
  npm start (dans le terminal du service)


PROBLÈME: MySQL non accessible
────────────────────────────────
Vérification:
  mysql -u root -e "SELECT 1;"

Si erreur:
  net start MySQL80 (démarrer MySQL)
  OU
  Chercher MySQL dans Services Windows et démarrer


PROBLÈME: Frontend timeout (ERR_CONNECTION_REFUSED)
─────────────────────────────────────────────────────
Solution:
  1. Arrêter Next.js (Ctrl+C)
  2. Relancer avec Turbopack désactivé:
     set NEXT_DISABLE_TURBOPACK=1
     npm run dev


PROBLÈME: npm install échoue
──────────────────────────────
Solution:
  1. Supprimer node_modules:
     rmdir /s node_modules
  2. Supprimer package-lock.json
  3. Réinstaller:
     npm install --force


PROBLÈME: Login erreur 401 ou 404
───────────────────────────────────
Vérifications:
  ✓ Auth Service est démarré (port 3002)
  ✓ API Gateway est démarré (port 3001)
  ✓ MySQL est actif
  ✓ BD auth_db existe

Vérifiez les logs dans le terminal de Auth Service


PROBLÈME: Les services ne démarrent pas du tout
─────────────────────────────────────────────────
Consultez:
  1. Le terminal (y a-t-il un message d'erreur?)
  2. Vérifiez le fichier .env du service
  3. Vérifiez que MySQL est actif
  4. Vérifiez que les ports ne sont pas utilisés
